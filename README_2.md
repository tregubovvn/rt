# Reading Test Editor

Одностраничное веб-приложение для создания и редактирования XML-тестов по чтению. Приложение предоставляет удобный графический интерфейс для работы с фрагментами текста и вопросами, а также интеграцию с GitHub Gist для хранения данных и DeepSeek AI для генерации контента.


## Особенности

- **Чистый интерфейс**: Современный темный дизайн без лишних визуальных шумов.
- **Двусторонняя синхронизация**: Изменения в форме мгновенно отражаются в XML, а ручное редактирование XML применяется к форме.
- **Работа с API**: Встроенная поддержка GitHub Gist для сохранения/загрузки и DeepSeek для AI-ассистенции.
- **Полная автономность**: Приложение написано на чистом HTML/CSS/JS, не требует сборщиков и серверной части. Достаточно открыть файл в браузере.
- **Два типа вопросов**: Поддержка вопросов с выбором ответа (Choice) и вопросов со свободным вводом (Input).

## Функциональность

### 1. Редактор фрагментов
Основная рабочая область приложения разбита на логические блоки:

- **Навигация**: Переключение между фрагментами теста с помощью вкладок или кнопок "Prev/Next". Возможность добавлять новые фрагменты и удалять существующие.
- **Настройки времени**: Поля `Time Excellent` и `Time Good` для установки временных нормативов выполнения.
- **Текст фрагмента**: Многострочное поле для ввода основного текста, который должен прочитать пользователь.

### 2. Редактор вопросов
Для каждого фрагмента можно создать неограниченное количество вопросов двух типов:

- **Choice (Выбор)**: 
  - Добавление вариантов ответов.
  - Выбор правильного ответа кликом на радио-кнопку.
  - Минимум 2 варианта ответа.
- **Input (Ввод)**:
  - Поле для указания правильного текстового ответа.

### 3. XML Панель
Нижняя часть экрана содержит код генерируемого XML.

- **Синхронизация**: XML обновляется в реальном времени при редактировании формы.
- **Apply Changes**: Кнопка парсинга вручную внесенных изменений в XML-код и применения их к визуальному редактору.
- **Копирование**: Быстрое копирование XML в буфер обмена.
- **Изменение размера**: Границу между редактором и XML-панелью можно перетаскивать мышью.

### 4. Интеграция с GitHub Gist
Приложение использует GitHub Gist как облачное хранилище для XML-файлов.

- **Save Gist**: Сохраняет текущее состояние теста. При первом сохранении создается новый Gist, при последующих — обновляется существующий.
- **Raw URL**: После сохранения в нижней строке состояния появляется прямая ссылка на raw-файл XML, которую можно скопировать и использовать для загрузки в других системах.
- **Load Gist**: Загрузка теста по ID Gist.

### 5. Интеграция с DeepSeek AI
Функция "Ask DeepSeek" позволяет отправить содержимое XML-панели в нейросеть DeepSeek.

- **Workflow**: Вы заполняете XML текстом или инструкциями (например, "Сгенерируй 3 вопроса к этому тексту..."), нажимаете кнопку, и AI возвращает результат прямо в XML-панель.
- **Гибкость**: Вы можете использовать промпты прямо в тексте XML, так как модель принимает всё содержимое поля как контекст.

## Настройка и Установка

### Шаг 1: Получение токенов

Для работы функций сохранения и AI вам понадобятся API-ключи.

**GitHub Token (Fine-grained Personal Access Token):**
1. Перейдите в [настройки токенов GitHub](https://github.com/settings/tokens?type=beta).
2. Нажмите **Generate new token (fine-grained)**.
3. Укажите имя токена и срок действия.
4. В разделе **Repository access** выберите "Only select repositories" (если хотите ограничить) или "All repositories".
5. В разделе **Permissions -> Account permissions** найдите **Gists** и установите **Read and write**.
6. Сгенерируйте токен и скопируйте его.

**DeepSeek Token:**
1. Зарегистрируйтесь или войдите на [platform.deepseek.com](https://platform.deepseek.com/).
2. Перейдите в раздел [API Keys](https://platform.deepseek.com/api_keys).
3. Создайте новый ключ и скопируйте его.

### Шаг 2: Запуск

1. Скачайте `index.html` из репозитория.
2. Откройте его в любом современном браузере.
3. Введите полученные токены в соответствующие поля в шапке приложения. Они сохранятся в `localStorage` вашего браузера.

## Формат XML

Приложение работает со следующей структурой XML:

```xml
<readingTest>
    <fragment timeExcellent="60" timeGood="90">
        <text>Текст фрагмента для чтения...</text>
        <questions>
            <question type="choice">
                <text>Текст вопроса?</text>
                <options>
                    <option>Неправильный ответ</option>
                    <option correct="true">Правильный ответ</option>
                </options>
            </question>
            <question type="input">
                <text>Вопрос со свободным ответом</text>
                <answer>Правильный ответ</answer>
            </question>
        </questions>
    </fragment>
</readingTest>
```

## Технологии

- **HTML5 / CSS3**: Семантическая верстка, CSS Variables для темизации, Flexbox/Grid для верстки.
- **Vanilla JavaScript**: Никаких фреймворков, только нативные API (Fetch, DOMParser, LocalStorage).
- **API**: GitHub REST API v3, DeepSeek Chat Completions API.
