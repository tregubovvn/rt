# RE:AD — Система Активного Чтения

**RE:AD** — это веб-приложение для осмысленного изучения текста, разработанное для образовательных целей. Приложение помогает студентам глубже погружаться в материал, используя методику поэтапного чтения с промежуточным и итоговым контролем знаний.

## 🚀 Основные возможности

1.  **Поэтапное чтение**: Текст разбивается на фрагменты, что облегчает усвоение больших объемов информации.
2.  **Тайминг чтения**: Визуальный индикатор времени помогает студенту следить за скоростью чтения.
    *   🟢 **Зеленый**: Оптимальное время (оценка "Отлично").
    *   🟡 **Желтый**: Допустимое время (оценка "Хорошо").
    *   🔴 **Красный**: Время превышено.
3.  **Два этапа тестирования**:
    *   **Промежуточный тест**: Проверка знаний сразу после прочтения фрагмента. Допускает повторные попытки, но ограничивает количество ошибок.
    *   **Финальный тест**: Проверка знаний по всему тексту. Одна попытка на вопрос, ошибки влияют на итоговую оценку.
4.  **Адаптивный дизайн**: Приложение корректно отображается на десктопах, планшетах и смартфонах.
5.  **Загрузка данных**: Поддержка загрузки заданий через XML-файл или напрямую через URL-параметр.
6.  **Современный интерфейс**: Стильный дизайн с анимациями и эффектом "Glassmorphism".

## 🎬 Демонстрация работы

1.  На стартовом экране студент вводит ФИО и загружает файл с заданием.
2.  После нажатия "Начать" открывается режим чтения с таймером.
3.  По кнопке "Готов к вопросам" начинается тест.
4.  При 4 ошибках в промежуточном тесте студент возвращается к чтению фрагмента.
5.  После успешного прохождения всех фрагментов запускается финальный тест.
6.  По итогам формируется ведомость с результатами.

## 💻 Установка и запуск

Приложение не требует установки сервера или баз данных. Это одностраничное приложение (SPA).

### Вариант 1: Локальный запуск
1.  Скачайте файл `index.html`.
2.  Откройте его в любом современном браузере (Chrome, Firefox, Edge, Safari).
3.  Используйте кнопку "Загрузить демо" для проверки или выберите свой XML-файл.

### Вариант 2: Размещение на сервере
Просто разместите файл `index.html` на вашем веб-сервере или хостинге статики.

### Вариант 3: Загрузка через URL
Вы можете передать ссылку на XML-файл через URL-параметр `file`. Приложение автоматически загрузит задание при старте.

Пример:
`https://ваш-сайт.com/index.html?file=https://example.com/task.xml`

## 📄 Формат файла задания (XML)

Задания для приложения представляют собой XML-файл со следующей структурой:

```xml
<readingTest>
    <!-- Фрагмент 1 -->
    <fragment timeExcellent="30" timeGood="60">
        <text>Текст первого фрагмента для чтения. Рекомендуемый объем около 100 слов.</text>
        <questions>
            <!-- Вопрос с выбором ответа -->
            <question type="choice">
                <text>Текст вопроса?</text>
                <options>
                    <option>Неправильный вариант</option>
                    <option correct="true">Правильный вариант</option>
                    <option>Еще один неправильный</option>
                </options>
            </question>
            
            <!-- Вопрос со свободным вводом -->
            <question type="input">
                <text>Напишите точный ответ (одно слово).</text>
                <answer>ключевое_слово</answer>
            </question>
        </questions>
    </fragment>

    <!-- Фрагмент 2 -->
    <fragment timeExcellent="45" timeGood="90">
        <text>Текст второго фрагмента...</text>
        <questions>
            <!-- ... вопросы ... -->
        </questions>
    </fragment>
</readingTest>
```

### Описание тегов и атрибутов:

| Тег/Атрибут | Описание |
| :--- | :--- |
| `<readingTest>` | Корневой элемент. |
| `<fragment>` | Контейнер для одного фрагмента текста и его вопросов. |
| `timeExcellent` | Время (в секундах) для получения оценки "Отлично" по таймеру. |
| `timeGood` | Время (в секундах) для получения оценки "Хорошо". Если время превышено — индикатор красный. |
| `<text>` | Содержимое фрагмента текста. |
| `<questions>` | Контейнер для списка вопросов к данному фрагменту. |
| `<question>` | Отдельный вопрос. Атрибут `type` может быть `choice` (выбор) или `input` (ввод). |
| `<options>` | Список вариантов ответов (для `type="choice"`). |
| `<option>` | Вариант ответа. Атрибут `correct="true"` помечает правильный ответ. |
| `<answer>` | Правильный ответ (для `type="input"`). Регистр букв при проверке не учитывается. |

## ⚙️ Логика работы приложения

### 1. Режим чтения
*   Студент видит текст и таймер.
*   Таймер меняет цвет в зависимости от установленных норм (`timeExcellent`, `timeGood`).
*   Время чтения сохраняется для статистики.

### 2. Промежуточное тестирование (Обучение)
*   Запускается сразу после прочтения фрагмента.
*   Вопросы выводятся по одному.
*   **Правила**:
    *   Если ответ неверный, студенту предлагается попробовать снова.
    *   Счетчик ошибок отображается в шапке (Ошибки: X/4).
    *   **Лимит ошибок**: Если студент ошибается 4 раза при ответе на вопросы к одному фрагменту, он принудительно возвращается к режиму чтения этого фрагмента.
    *   При правильном ответе происходит переход к следующему вопросу.

### 3. Итоговый тест
*   Запускается после прохождения всех фрагментов.
*   Включает вопросы ко всем фрагментам подряд.
*   **Правила**:
    *   Одна попытка на каждый вопрос.
    *   Правильный/неправильный ответ подсвечивается, но переход к следующему вопросу происходит автоматически.
    *   Повторные попытки невозможны.
    *   Все ответы сохраняются для итогового отчета.

### 4. Результаты
*   Выводится итоговая оценка.
*   Отображается общее время чтения.
*   Выводится таблица ответов **только по финальному тесту**.
*   Промежуточные ошибки (если они были исправлены) в отчет не попадают.

## 📊 Система оценивания

Итоговая оценка формируется на основе количества ошибок в **финальном тесте**:

*   **Оценка 5**: 0 ошибок.
*   **Оценка 4**: 1-2 ошибки.
*   **Оценка 3**: 3 ошибки.
*   **Оценка 2**: 4 и более ошибок.

*(Примечание: Время чтения влияет на визуальную индикацию, но в текущей версии не снижает итоговую оценку автоматически, что позволяет преподавателю гибко подходить к оцениванию).*

## 🛠 Технические детали

*   **Технологии**: HTML5, CSS3 (Flexbox, Grid, Variables), Vanilla JavaScript (ES6+).
*   **Стилизация**: Адаптивная верстка с использованием `clamp()` для плавного масштабирования шрифтов.
*   **Парсинг данных**: Используется встроенный `DOMParser` для обработки XML.
*   **Безопасность**: Приложение работает на стороне клиента (Client-side), данные ФИО не передаются на сервер.
